# 🤖 XP Bot — Telegram бот для гейміфікації спільноти

Telegram-бот на Python для мотивації учасників каналу/чату через систему досвіду (XP), завдань, магазину нагород та таблиці лідерів.

---

## ✨ Можливості

### 👤 Для користувачів
| Команда | Опис |
|---|---|
| `/start` | Реєстрація та привітання |
| `/tasks` | Переглянути список активних завдань |
| `/xp` | Мій профіль: XP, ранг, статистика |
| `/leaderboard` | Таблиця лідерів спільноти |
| `/shop` | Магазин — витрачай XP на нагороди |
| `/inventory` | Мій інвентар (куплені товари) |
| `/help` | Як працює бот |

### 🛡️ Для адміністраторів
| Команда | Опис |
|---|---|
| `/admin` | Головна панель адміна |
| `/addtask` | Додати нове завдання |
| `/deltask` | Видалити завдання |
| `/givexp` | Видати або забрати XP у користувача |
| `/addproduct` | Додати товар до магазину |
| `/editproduct` | Редагувати товар |
| `/delproduct` | Видалити товар |
| `/stats` | Статистика бота |
| `/ban` / `/unban` | Бан/розбан користувача |

---

## ⚙️ Як працює

1. Адмін створює **завдання** (наприклад: "Зробити пост", "Запросити друга") з винагородою в XP
2. Користувач виконує завдання та надсилає **підтвердження** боту
3. Адмін **схвалює або відхиляє** виконання
4. Після схвалення — XP нараховується автоматично
5. XP можна витратити в **магазині** на реальні або цифрові нагороди
6. **Таблиця лідерів** мотивує учасників змагатися

## 🚀 Встановлення

### Вимоги
- Python 3.11+
- Telegram Bot Token (отримати у [@BotFather](https://t.me/BotFather))

### Кроки

```bash
# 1. Клонувати репозиторій
git clone https://github.com/andrew-buga/xp-bot.git
cd xp-bot

# 2. Створити віртуальне середовище
python -m venv .venv
.venv\Scripts\activate  # Windows
# або
source .venv/bin/activate  # Linux/Mac

# 3. Встановити залежності
pip install -r requirements.txt

# 4. Налаштувати змінні середовища
copy .env.example .env   # Windows
# або
cp .env.example .env     # Linux/Mac

# Відкрити .env та заповнити:
# BOT_TOKEN=ваш_токен_від_botfather
# ADMIN_IDS=ваш_telegram_id

# 5. Запустити бота
python bot.py
```

---

## 🔒 Безпека
- Токен зберігається у `.env` (не в коді)
- `.env` файл не потрапляє у git
- Вбудований anti-flood захист (макс. 8 запитів / 10 сек на користувача)

---

## 🖥️ Розгортання на сервері (Linux VPS)

Готові скрипти у папці `scripts/`:

```bash
# Автоматичне встановлення як системного сервісу
bash scripts/bootstrap_server.sh
bash scripts/install_service.sh
```

Бот запускається як `systemd`-сервіс і автоматично стартує після перезавантаження сервера.

## 💡 Запуск двох копій на одному сервері

Хочете запустити бота для двох різних спільнот без додаткового хосту?

**Linux/Mac:**
```bash
cp -r /path/to/xp-bot /path/to/xp-bot-2
cd /path/to/xp-bot-2
cp .env.example .env
# Заповнити BOT_TOKEN та ADMIN_IDS для другого бота
python bot.py
```

**Windows (PowerShell):**
```powershell
Copy-Item -Path "C:\xp-bot" -Destination "C:\xp-bot-2" -Recurse
cd C:\xp-bot-2
# Відредагувати .env — вставити новий токен
notepad .env
.venv\Scripts\Activate.ps1
python bot.py
```

Кожен екземпляр працює незалежно зі своєю базою даних (`bot_data.db`).
